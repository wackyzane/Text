Mongoose Explained | By: Perry Karlsen 03/01/19

Mongoose

Downloading Mongoose

1. First check to see if you have mongoose. With npm list -g mongoose. You know if you have it by the version near the end of the path. For me the version is v10.15.0. If you don't have it you can install it with npm install mongoose.

Mongoose Schema

2. A schema is a method like in computer science. Once you call upon it, the method will run top to bottom and set any variables, print and messages, and run anything you tell it too. You can call upon Schemas inside Schemas.

Running Mongoose In Node

3. You can run the server by opening a command console in the folder with app.js. Then in the Command console type in node app.js and it will open up in a browser with the address being localhost:8080.

Mongoose Books

4. In the code we made a function that finds all the books in our database and prints them onto the page for you to see. If you are using chrome it won't be in the pretty format you see. I'm using firefox for the looks. Make sure to see the URL since it won't show books unless you have the /books at the end.

Mongoose One Book

5. We can print one book at a time also, though it requires more code. In the picture showing the code you can see it bringing up the code along with asking for an ID using /:id. So under that is the method where we use the ID provided in the URL. Notice in the Page picture's URL that I had to copy and paste the Book ID afters books.

Postman

6. Postman is a software we tried out that made our books for us. There were a couple steps we needed to do first such as, type in our URL, go onto the body tab, circle in the x-www-form-urlencoded and from there we could type in our titles and there values. Lastly we press send to add them.
